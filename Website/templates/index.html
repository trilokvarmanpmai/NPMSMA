🧩 Docs — Call NPM-SMA Without the UI

The web studio above is just one client. Underneath it, NPM-SMA is a plain HTTP endpoint: send a video and one or more platform authorization codes as a multipart form, and it does the rest — AI metadata, upload and publishing. Python, JavaScript, Java, or literally anything that can send a POST request can talk to it directly.

POST /upload 
import requests

url = "http://localhost:5000/upload"

files = {
    "video": open("my_video.mp4", "rb"),
    "thumbnail": open("thumb.jpg", "rb"),  # optional
}
data = {
    "auth_code_facebook": "AUTH_CODE_FROM_FACEBOOK_OAUTH",
    "auth_code_youtube":  "AUTH_CODE_FROM_YOUTUBE_OAUTH",
}

response = requests.post(url, files=files, data=data, timeout=1200)
print(response.json())
const form = new FormData();
form.append("video", videoFile);          // File object
form.append("thumbnail", thumbnailFile);  // optional
form.append("auth_code_instagram", authCode);
form.append("auth_code_linkedin", otherAuthCode);

const res = await fetch("http://localhost:5000/upload", {
  method: "POST",
  body: form,
});
const result = await res.json();
console.log(result);
HttpClient client = HttpClient.newHttpClient();

var bodyPublisher = MultipartBodyPublisher.newBuilder()
    .addPart("video", Path.of("my_video.mp4"))
    .addPart("auth_code_tiktok", "AUTH_CODE_FROM_TIKTOK_OAUTH")
    .build();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("http://localhost:5000/upload"))
    .header("Content-Type", bodyPublisher.contentType())
    .POST(bodyPublisher)
    .build();

HttpResponse<String> response =
    client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
video

Required. The video file to publish.

thumbnail

Optional image file, used where the platform supports a custom thumbnail.

auth_code_facebook

OAuth code from Facebook's own authorization screen.

auth_code_instagram

OAuth code from Instagram's authorization screen.

auth_code_thread

OAuth code from Threads' authorization screen.

auth_code_linkedin

OAuth code from LinkedIn's authorization screen.

auth_code_tiktok

OAuth code from TikTok's authorization screen.

auth_code_youtube

OAuth code from YouTube's authorization screen.

Only send codes for the platforms you want to post to — any missing field is simply skipped. The response mirrors the publishing pipeline's own JSON reply for each connected platform.
